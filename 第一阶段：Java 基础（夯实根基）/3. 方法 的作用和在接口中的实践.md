明白，我们来详细讲 **方法** 的作用和在接口中的实践，特别是 **参数、返回值、重载**，以及如何实现统一结果返回。

---

## 1️ 方法基础概念

方法是**类的行为**，用来封装可重复执行的逻辑。

```java
class Calculator {
    // 方法：计算两个整数和
    public int add(int a, int b) {
        return a + b;
    }
}
```

### 核心点

| 概念  | 作用         | 示例                  |
| --- | ---------- | ------------------- |
| 参数  | 外部传入方法的数据  | `int a, int b`      |
| 返回值 | 方法执行后返回的结果 | `int`               |
| 方法名 | 唯一标识       | `add`               |
| 方法体 | 执行逻辑       | `{ return a + b; }` |

---

## 2️ 方法重载（Overload）

**重载**是同一个类里，方法名相同，但参数列表不同。

```java
class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

> **应用场景**：接口方法可以支持不同参数类型或数量，提升灵活性。

---

## 3️ 封装结果类（统一返回）

在接口开发中，通常会用一个 **Result** 类来统一返回格式：

```java
class Result<T> {
    private int code;
    private String message;
    private T data;

    // 构造器
    public Result(int code, String message, T data) {
        this.code = code;
        this.message = message;
        this.data = data;
    }

    // 静态方法快速创建
    public static <T> Result<T> success(T data) {
        return new Result<>(200, "success", data);
    }

    public static <T> Result<T> error(String message) {
        return new Result<>(500, message, null);
    }

    // getter/setter
    public int getCode() { return code; }
    public String getMessage() { return message; }
    public T getData() { return data; }
}
```

> **作用**：所有接口返回统一结构，便于前端解析。

---

## 4️ 计算类示例（封装方法 + 调用 Result）

```java
class CalculatorService {

    // 两数相加
    public Result<Integer> add(int a, int b) {
        int sum = a + b;
        return Result.success(sum);
    }

    // 三数相加（重载）
    public Result<Integer> add(int a, int b, int c) {
        int sum = a + b + c;
        return Result.success(sum);
    }

    // 除法，带异常处理
    public Result<Double> divide(double a, double b) {
        if (b == 0) {
            return Result.error("除数不能为0");
        }
        return Result.success(a / b);
    }
}
```

## 5️ Controller 调用示例

```java
@RestController
@RequestMapping("/calc")
public class CalculatorController {

    private final CalculatorService service;

    public CalculatorController(CalculatorService service) {
        this.service = service;
    }

    @GetMapping("/add")
    public Result<Integer> add(@RequestParam int a, @RequestParam int b) {
        return service.add(a, b);
    }

    @GetMapping("/add3")
    public Result<Integer> add3(@RequestParam int a, @RequestParam int b, @RequestParam int c) {
        return service.add(a, b, c); // 方法重载调用
    }

    @GetMapping("/divide")
    public Result<Double> divide(@RequestParam double a, @RequestParam double b) {
        return service.divide(a, b);
    }
}
```

## 6️ 流程总结（方法 + 接口调用）

1. Controller 接收请求参数 → 封装成方法参数
2. 调用 Service 的方法 → 执行业务逻辑
3. 方法返回 Result 对象 → Controller 直接返回给前端
4. 前端拿到统一格式数据，方便解析

```
前端请求 → Controller → Service.method(params) → Result 返回 → Controller → HTTP 响应
```

>  **要点**：
>
> * **方法参数**：接收输入
> * **方法返回值**：输出数据
> * **方法重载**：支持多种参数组合
> * **统一结果封装**：支撑接口规范化返回
