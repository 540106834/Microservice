# 一、项目核心目标（先搞清本质）

本质就一句话：

👉 **记录学生什么时候进校、什么时候离校，并保证家长 / 学校能看到。**

听起来简单，但里面其实藏着：

* 高并发（早上同时刷卡）
* 数据安全
* 实时通知
* 设备接入
* 可观测性

已经很接近真实互联网系统了。

---

# 二、系统功能设计（推荐版本 ⭐⭐⭐⭐⭐）

## 1️⃣ 基础功能（必须做）

### ✅ 学生管理

* 添加学生
* 班级绑定
* 家长手机号

### ✅ 考勤记录

记录：

```
学生ID
时间
类型（上学 / 放学）
设备ID
```

---

### ✅ 刷卡 / 人脸 / 二维码

模拟即可：

比如：

```
POST /checkin
```

参数：

```json
{
  "studentId": 1001,
  "type": "IN"
}
```

---

### ✅ 家长通知（非常加分🔥）

学生进校：

> 【XX学校】您的孩子已于 07:32 安全入校。

实现方式：

* 初级：打印日志
* 进阶：接入短信 API
* 高级：消息队列异步发送

---

## 2️⃣ 推荐加的高级功能（拉开水平）

### ⭐ 实时考勤大屏

统计：

* 今日到校人数
* 未到校名单
* 提前离校

👉 运维面试官超爱问这种。

---

### ⭐ 异常检测

例如：

* 9:00 还没来 → 标记异常
* 未刷进校直接刷离校

可以写定时任务：

```
@Scheduled
```

---

### ⭐ 老师后台

查看：

* 班级出勤率
* 迟到名单

---

# 五、核心代码逻辑（超简单但非常经典）

### 打卡逻辑：

伪代码：

```
1. 判断学生是否存在
2. 判断是否重复打卡
3. 写入数据库
4. 发送通知
```

---

### 一个隐藏难点 ⭐⭐⭐⭐⭐

## ❗如何防止重复打卡？

早上学生可能狂刷卡 😂

解决：

### 方法1（推荐）

Redis：

```
key:
studentId + date + type
```

存在 → 拒绝

---

### 方法2

数据库唯一索引：

```
(student_id, date, type)
```

双保险。
