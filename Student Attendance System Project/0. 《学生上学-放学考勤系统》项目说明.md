## 一、项目背景与目标

随着校园信息化推进，传统人工点名、纸质登记在**效率、准确性、追溯性**方面逐渐暴露短板。
本项目旨在建设一套**覆盖幼儿园、小学、初中、高中**的统一学生考勤系统，实现学生上下学考勤的数字化、标准化与可追溯化。

**核心目标：**

* 实现学生**上学 / 放学**自动考勤
* 支持**多学段、多学校、多区域**统一管理
* 为班主任、学校、教育主管部门提供实时数据支撑
* 具备从**单体系统平滑演进到微服务**的能力

---

## 二、适用范围

* **地域范围**：省级（可下钻到市 / 区 / 校）
* **学校类型**：

  * 幼儿园
  * 小学
  * 初中
  * 高中
* **使用对象**：

  * 学生
  * 教师（班主任）
  * 学校管理员
  * 教育局管理人员

---

## 三、系统整体定位

> 这是一个**以“学生”为中心、以“考勤事件”为主线**的基础教育信息系统。

系统强调三点：

1. **数据真实**：考勤记录自动生成、不可随意篡改
2. **结构清晰**：统一编码规则，便于跨系统对接
3. **架构可成长**：从单体起步，不推翻、不重写

---

## 四、核心功能说明

### 1️⃣ 学生基础信息管理

* 学生唯一编码
* 姓名、性别、年级、班级
* 所属学校、区域信息

### 2️⃣ 学校与组织结构管理

* 区域 → 学校 → 年级 → 班级 四级结构
* 支持不同学段差异化配置

### 3️⃣ 考勤管理（核心）

* 上学打卡
* 放学打卡
* 自动生成考勤记录
* 状态识别：

  * 正常
  * 迟到
  * 早退
  * 缺勤

### 4️⃣ 考勤查询与统计

* 学生个人考勤记录
* 班级 / 年级 / 学校统计
* 日 / 周 / 月维度汇总

### 5️⃣ 权限与角色

* 学生：仅查看自身记录
* 教师：查看所带班级
* 管理员：全校管理
* 教育局：区域汇总视角

## 五、技术实现原则

### 架构策略

* **第一阶段**：单体 Spring Boot 应用
* **第二阶段**：按模块拆分（用户 / 考勤 / 学校）
* **第三阶段**：微服务化 + 注册中心 + 配置中心

### 技术选型（初始）

* 后端：Java + Spring Boot
* 数据库：MySQL
* ORM：MyBatis / JPA
* 接口风格：RESTful
* 日志：统一日志 + 审计日志
