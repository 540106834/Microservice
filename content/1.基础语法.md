好，这一节我们就**只学 Web / API 开发“够用即可”的 Java 基础语法**。
不绕弯，不刷题，不讲你以后用不到的语法糖。像给工具箱配常用扳手 

---

## 一、变量 & 类型（API 里最常见）

### 1️ 基本写法

```java
int age = 18;
long id = 1001L;
String name = "Tom";
boolean enabled = true;
```

 API 开发中最常用：

* `int / Integer`
* `long / Long`
* `String`
* `boolean / Boolean`

---

### 2️ 基本类型 vs 包装类型（Web 必懂）

```java
int a = 1;
Integer b = 1;
```

区别你只记一句话就够：

> **JSON / 请求参数 / 数据库 → 一律用包装类型**

为什么？

```java
Integer age = null; // 合法
int age = null;     // 编译错误
```

 Controller / DTO / Entity：

* 用 `Integer / Long`
* 少用 `int / long`

---

## 二、条件判断（接口分支逻辑）

### if / else

```java
if (age >= 18) {
    return "adult";
} else {
    return "child";
}
```

### 三目运算（少量用）

```java
String role = age >= 18 ? "USER" : "GUEST";
```

 Web 中：

* 复杂逻辑 → `if`
* 简单赋值 → 三目

---

## 三、循环（接口里很少写复杂的）

### for（最常见）

```java
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

### foreach（推荐）

```java
for (User user : users) {
    System.out.println(user.getName());
}
```

🚫 不要在 API 里写多重嵌套循环，性能雷区。

---

## 四、方法（接口就是方法）

### 基本结构

```java
public String sayHello(String name) {
    return "hello " + name;
}
```

结构拆解：

```
修饰符 返回类型 方法名(参数)
```

---

### void vs return

```java
public void log(String msg) {
    System.out.println(msg);
}

public int add(int a, int b) {
    return a + b;
}
```

📌 Controller 方法：

* 一定有返回值（JSON）

---

## 五、类（API 的最小单位）

### 一个普通 Java 类

```java
public class User {
    private Long id;
    private String name;
}
```

### Getter / Setter（JSON 关键）

```java
public Long getId() {
    return id;
}

public void setId(Long id) {
    this.id = id;
}
```

👉 没 getter/setter：

* JSON 反序列化失败
* 返回字段丢失

---

## 六、对象 & new（实例化）

```java
User user = new User();
user.setId(1L);
user.setName("Tom");
```

⚠️ 在 Spring 中：

* **少自己 new**
* 多交给 Spring 管理（@Autowired）

---

## 七、null（Web API 最大敌人）

### 常见 NPE 场景

```java
user.getName();
```

如果 user 是 null 💥

### 最低防线

```java
if (user == null) {
    throw new RuntimeException("user not found");
}
```

📌 Web API：

* 允许抛异常
* 不要静默失败

---

## 八、== vs equals（必须记）

```java
String a = new String("x");
String b = new String("x");

a == b        // false
a.equals(b)  // true
```

API 判断内容：

* **永远用 equals**

---

## 九、import & package（项目结构）

```java
package com.jsy.demo.controller;

import java.util.List;
```

📌 规则：

* package = 目录结构
* 一个类一个文件

## 十、一眼能看懂 Controller（目标）

看到下面代码你应该完全不慌：

```java
@GetMapping("/user/{id}")
public User getUser(@PathVariable Long id) {
    User user = userService.getById(id);
    if (user == null) {
        throw new RuntimeException("not found");
    }
    return user;
}
```

你现在应该能解释：

* 参数怎么来的
* 返回什么
* 异常什么时候抛
