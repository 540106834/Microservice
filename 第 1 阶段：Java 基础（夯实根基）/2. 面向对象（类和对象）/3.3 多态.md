
## 什么是多态？（一句话版）

> **同一句话，换个人来做，做法不同。**

命令不变，执行方式会变。

---

## 一个生活画面

你对不同人说同一句话：

> **“开始干活”**

* 对厨师 → 开始炒菜 🍳
* 对司机 → 开始开车 🚗
* 对程序员 → 开始敲代码 ⌨️

你没改指令，对方的**身份**决定了行为。

👉 这就是多态。

---

## 放到 Java 里

### 统一的“指令”（父类 / 接口）

```java
interface Service {
    void start();
}
```

### 不同的“人”（实现类）

```java
class WebService implements Service {
    public void start() {
        System.out.println("启动 Web 服务");
    }
}

class JobService implements Service {
    public void start() {
        System.out.println("启动 定时任务");
    }
}
```

### 你只说一句话

```java
Service s = new WebService();
s.start();   // 启动 Web 服务

s = new JobService();
s.start();   // 启动 定时任务
```

你没换命令，**结果自己变了**。

---

## 多态到底“多”在哪？

不是方法多
不是类多

而是：

> **同一个调用点 → 多种行为**

```java
s.start();
```

这一行，能干很多种事。

---

## 多态的作用（通俗版）

### ① 不用关心“是谁”，只关心“能不能干”

你只要求：

> 有 `start()`

不要求：

> 一定是 WebService

世界一下子松了。

---

### ② 新东西来了，不用改老代码

今天只有：

* WebService
* JobService

明天加一个：

```java
class CacheService implements Service {
    public void start() {
        System.out.println("启动缓存服务");
    }
}
```

原来的代码：

```java
s.start();
```

**一行不动，还能跑新服务**。

---

### ③ 少 if / else，少出事故

没有多态，你会写：

```java
if (type == WEB) { ... }
else if (type == JOB) { ... }
```

有多态：

```java
s.start();
```

代码更短，脑子更轻，事故更少。

---

## 一句话总结（记这个）

> **多态 = 同一个接口，不同的实现
> 作用 = 让代码不写死，能随时换人干活**

---

## 再压缩成“程序员黑话”

* 父类 / 接口：定规则
* 子类：干具体的事
* 调用方：只管喊一声
