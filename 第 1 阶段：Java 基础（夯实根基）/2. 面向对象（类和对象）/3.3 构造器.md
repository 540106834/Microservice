
### 1. 构造器是什么

构造器（constructor）是类的一种特殊方法，用来**初始化对象**。它的名字必须和类名完全一样，并且**没有返回值类型**（连 `void` 都不能写）。

例如：

```java
class Person {
    String name;

    // 构造器
    public Person(String name) {
        this.name = name;
    }
}
```

---

### 2. 构造器前可以加的修饰符

构造器可以使用访问修饰符，主要有以下几种：

| 修饰符         | 说明                       |
| ----------- | ------------------------ |
| `public`    | 对所有类可见，任何地方都可以创建对象。      |
| `protected` | 同一个包或者子类可以访问。            |
| （默认）        | 同一个包可以访问（包访问权限）。         |
| `private`   | 只能在类内部使用，常用于单例模式或工厂方法模式。 |

---

### 3. 例子对比

```java
class A {
    public A() {
        System.out.println("Public 构造器");
    }

    protected A(int x) {
        System.out.println("Protected 构造器");
    }

    A(String s) {
        System.out.println("默认访问构造器");
    }

    private A(double d) {
        System.out.println("Private 构造器");
    }
}

public class Test {
    public static void main(String[] args) {
        A a1 = new A();       // 可以
        A a2 = new A(5);      // 同包可以
        A a3 = new A("Hi");   // 同包可以
        // A a4 = new A(3.14); // 错误，private 构造器不能外部访问
    }
}
```

---

###  总结

* 构造器前面**可以有 `public`**，这只是控制它的访问权限。
* `public` 构造器意味着任何地方都能创建对象。
* 不写修饰符就是默认包访问权限，`private` 常用于控制实例化。
