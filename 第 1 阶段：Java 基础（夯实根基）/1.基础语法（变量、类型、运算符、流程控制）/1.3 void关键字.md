
## 一、`void` 是什么？

`void` 是 **方法的返回类型**，表示：

> 这个方法 **执行动作，但不返回任何结果**

最常见的出现场景：

```java
public static void main(String[] args) {
    System.out.println("Hello");
}
```

这里的 `main`：

* 会被 JVM 调用
* 会执行代码
* **不会返回值给调用者**

---

## 二、为什么一定要写 `void`？

Java 的方法签名必须明确：

```text
修饰符 + 返回类型 + 方法名 + 参数
```

如果你不写返回类型，Java 会直接拒绝编译 。

❌ 错误示例：

```java
public sayHello() {
    System.out.println("Hi");
}
```

✅ 正确示例：

```java
public void sayHello() {
    System.out.println("Hi");
}
```

---

## 三、`void` 方法里能不能 `return`？

可以，但只能是 **空 return**：

```java
public void check(int x) {
    if (x < 0) {
        return;   // 提前结束方法
    }
    System.out.println(x);
}
```

❌ 不允许：

```java
return 123; // 编译错误
```

---

## 四、`void` 和有返回值的方法对比

### 有返回值

```java
public int add(int a, int b) {
    return a + b;
}
```

### 无返回值

```java
public void printSum(int a, int b) {
    System.out.println(a + b);
}
```

**区别一句话版**：

* `int`：算完，把结果交出去
* `void`：算完，只做事，不交东西

---

## 五、`void` 能不能作为变量类型？

不能 ❌

```java
void x;   // 编译错误
```

原因很简单：

> `void` 不是一种“值类型”，而是“无返回”的标记

## 六、和你运维/服务端学习最相关的地方 

在你后面学 **Spring Boot / Controller** 时，会经常看到：

```java
@PostMapping("/reload")
public void reloadConfig() {
    configService.reload();
}
```

含义是：

* 接口被调用
* 执行一次操作
* **不关心返回数据，只关心是否成功**

当然，很多项目会改成返回 `Result` 或 `ResponseEntity`，这是设计选择，不是语法问题。
